# SMART Principle

**Origin:** George T. Doran (1981)

## The Criteria

| Letter | Meaning | Key Question |
|--------|---------|-------------|
| **S** | **Specific** | What exactly? Who? Where? |
| **M** | **Measurable** | How will I know it's done? |
| **A** | **Achievable** | Is it realistic with available resources? |
| **R** | **Relevant** | Does it align with broader goals? |
| **T** | **Time-bound** | When must it be completed? |

## Examples

| ❌ Vague | ✅ SMART |
|----------|---------|
| "Get more customers" | "Acquire 50 new enterprise customers in Q3 2026" |
| "Improve code quality" | "Achieve 90% test coverage on all new PRs by Dec 2026" |
| "Learn Python" | "Complete 3 Python projects and pass PCEP exam by June 2026" |
| "Update the website" | "Redesign landing page and reduce bounce rate from 60% to 40% by Aug 2026" |

## When to Use
- Defining project goals and milestones
- Individual performance objectives
- Sprint or quarterly planning
- Any goal that needs clear success criteria

## Pitfalls
- ❌ Being too rigid — SMART works best for short-to-medium term goals
- ❌ "Achievable" used as excuse to set low bars
- ❌ Focusing on metrics (M) so much that you lose the big picture (R)
