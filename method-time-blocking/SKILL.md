# Time Blocking (时间块管理)

**Popularized by:** Cal Newport, *Deep Work* (2016) — but exists in various forms for decades

**Core idea:** Plan every minute of your day by assigning specific tasks to specific blocks on your calendar. No "wake up and figure it out."

## The Method

### Step 1: Identify Block Types

| Block Type | Duration | Example |
|------------|----------|---------|
| **Deep work** | 2–4 hours | Feature development, writing, design |
| **Shallow work** | 30–60 min blocks | Email, Slack, status updates |
| **Meetings** | As scheduled | Standups, reviews, 1:1s |
| **Buffer** | 30–60 min | Overflow, unexpected tasks |
| **Recovery** | 15–30 min | Breaks, walk, lunch |

### Step 2: Build Your Ideal Week Template

Example:
```
Mon 09:00–12:00  Deep work (project A)
Mon 14:00–15:00  Shallow work / email
Mon 15:00–16:00  Team sync
Tue 09:00–11:00  Deep work (project B)
...
```

### Step 3: Execute With Discipline
- Treat each block like an appointment with yourself
- No multitasking during a block
- If interrupted, move the block — don't drop it
- At end of day, plan tomorrow's blocks

## Time Boxing (variant)
Set an upper limit on a task. If the timebox expires, stop and reassess.

**Time blocking:** "I'll write from 9–11" (you do whatever it takes)
**Timeboxing:** "I'll spend 2 hours writing" (strict shutdown)

## Common Mistakes
- ❌ Not leaving buffer time (impossible to predict everything)
- ❌ Bouncing between tasks within a block
- ❌ Blocking too granularly (<15 min)
- ❌ Treating blocks as optional (let others hijack your calendar)

## When to Use
- Day feels reactive, not proactive
- Too much context switching
- Important work keeps getting pushed to "later"
- You're in back-to-back meetings with no breathing room
