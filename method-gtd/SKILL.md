# GTD (Getting Things Done)

**Origin:** David Allen, *Getting Things Done* (2001)

Core principle: **Your brain is for having ideas, not holding them.** Externalize everything into a trusted system.

## Five-Step Workflow

### 1. Collect
Capture everything that has your attention into an inbox:
- Physical inbox (notebook, index cards)
- Digital inbox (email, notes app, task manager)
- Voice memos

No filtering or judging at this stage.

### 2. Process

Apply the **2-minute rule**: if it takes ≤2 minutes, do it now.

```
Is it actionable?
├── YES → Takes ≤2 min? → Do it now
│         Takes >2 min? → Delegate or Defer
└── NO  → Reference? → File it
          Someday/maybe? → Separate list
          Neither? → Delete
```

### 3. Organize

| Category | Description |
|----------|-------------|
| **Next Actions** | The very next physical step for each project |
| **Projects** | Outcomes requiring more than one step |
| **Waiting For** | Tasks dependent on others |
| **Calendar** | Time-specific appointments only |
| **Someday/Maybe** | Ideas for future consideration |

### 4. Review
- **Daily:** Process inbox to zero, check calendar and next actions
- **Weekly:** Full review of all projects, next actions, waiting fors, someday/maybe

### 5. Engage
Choose what to do by: **Context** → **Time available** → **Energy level** → **Priority**

## Common Tools
- Digital: Todoist, Things 3, Omnifocus, Notion
- Analog: Bullet Journal, index cards
- Minimal: Single text file with sections

## When to Use
- You feel overwhelmed by too many tasks floating in your head
- Tasks keep falling through the cracks
- You want a single trusted system for everything
- Weekly or daily planning sessions feel chaotic
