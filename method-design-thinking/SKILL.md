# Design Thinking

**Origin:** Stanford d.school / IDEO (Tim Brown, David Kelley)

**Core idea:** Solve complex problems through deep human understanding, iterative prototyping, and rapid feedback — putting the user at the center.

## The Five Phases

```
  Empathize → Define → Ideate → Prototype → Test ↻
                                                        │
                                                        └── (iterate)
```

| Phase | Goal | Activities |
|-------|------|------------|
| 1. **Empathize** | Understand the user | Interviews, observation, immersion |
| 2. **Define** | Frame the problem | Synthesis, point-of-view statement, how-might-we |
| 3. **Ideate** | Generate possibilities | Brainstorming, SCAMPER, worst possible idea |
| 4. **Prototype** | Make ideas tangible | Low-fi sketches, paper prototypes, role-playing |
| 5. **Test** | Validate with users | Usability testing, feedback capture, iterate |

## Mindset Principles

| Principle | Meaning |
|-----------|---------|
| **Human-centered** | Start with real user needs, not technology |
| **Radical collaboration** | Cross-disciplinary teams (designers, engineers, PMs) |
| **Iterate, iterate, iterate** | Fail early, fail often, learn fast |
| **Bias toward action** | Build to think, don't just talk |
| **Show, don't tell** | Visualize ideas, create tangible artifacts |

## Common Tools per Phase

| Phase | Tools |
|-------|-------|
| Empathize | Empathy map, user journey map, persona, interview guide |
| Define | Point-of-view (POV) statement, problem statement, user story |
| Ideate | Brainstorming, mind map, crazy 8s, SCAMPER |
| Prototype | Paper prototype, wireframe, clickable mockup, role-play |
| Test | Usability test, A/B test, feedback grid |

## When to Use
- Designing new products or services
- Solving ambiguous, people-centered problems
- Exploring new markets or user segments
- Any situation where user needs are poorly understood
