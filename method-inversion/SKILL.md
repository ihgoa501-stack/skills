# Inversion Thinking (逆向思维)

**Origin:** Stoic philosophy (Seneca, *On Anger*) → modern popularization by Charlie Munger

**Core idea:** Instead of asking "How do I achieve success?", ask **"What would guarantee failure?"** — then avoid those things.

## Two Forms

### 1. Avoid the Negative
If the goal is "improve project delivery," invert:

| Forward | Inverse |
|---------|---------|
| "What makes a project successful?" | "What would make it fail?" |
| Strong leadership | No clear ownership |
| Good communication | Siloed info, no standups |
| Realistic deadlines | Scope creep without renegotiation |

**Result:** Focus on eliminating failure modes.

### 2. Consider the Opposite
Before committing to a decision:
- **Pre-mortem:** "It's 1 year later and this project failed miserably. What went wrong?"
- Challenge assumptions by imagining the worst-case scenario

## Usage Examples

- **Investing:** "What would lose me money?" → avoid those behaviors
- **Product design:** "What would make users hate this?" → remove friction points
- **Strategy:** "Which competitors would kill us if we did X?" → assess risk

## When to Use
- Over-optimistic planning
- Confirmation bias setting in
- Need to identify blind spots
- High-stakes decisions where downside matters more than upside
