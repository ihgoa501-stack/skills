# Cynefin Framework (Cynefin 框架)

**Origin:** Dave Snowden, IBM Global Services (1999), later refined into Cognitive Edge

## The Five Domains

| Domain | Causal Relationship | Action |
|--------|-------------------|--------|
| **Clear** (Simple) | Cause → effect is obvious | Sense → Categorize → Respond (best practice) |
| **Complicated** | Cause → effect exists but requires expertise | Sense → Analyze → Respond (good practice) |
| **Complex** | Cause → effect only visible in retrospect | Probe → Sense → Respond (emergent practice) |
| **Chaotic** | No perceivable cause → effect | Act → Sense → Respond (novel practice) |
| **Disorder** | Don't know which domain you're in | Break the situation into parts and assign domains |

## How to Use

1. **Identify which domain** your problem lives in
2. **Match your approach** to the domain — the biggest mistake is using a Clear-domain solution on a Complex problem
3. **Reassess** as the situation evolves

## Domain-Specific Pitfalls

| Domain | Danger |
|--------|--------|
| Clear | Complacency — slipping into "we've always done it this way" |
| Complicated | Analysis paralysis — over-studying, under-acting |
| Complex | Premature convergence — locking into one answer too soon |
| Chaotic | Addictive command-and-control — failing to delegate once stability returns |

## Key Insight

> **Most real-world problems are Complex, but most management training assumes Clear or Complicated.**

Applying best-practice thinking to a Complex problem leads to fragile solutions that break under change.

## When to Use

- You're trying a solution that works elsewhere but failing — you may be in the wrong domain
- Leading through uncertainty or crisis
- Designing organizational processes — match governance to problem type
- Before choosing methodology — Agile, Lean, and Six Sigma each excel in different domains
