# PDCA Cycle (戴明环 / Plan-Do-Check-Act)

**Origin:** Walter Shewhart → W. Edwards Deming

## The Cycle

```
  ┌──────────┐
  │  PLAN    │
  └────┬─────┘
       ↓
  ┌──────────┐
  │   DO     │
  └────┬─────┘
       ↓
  ┌──────────┐
  │  CHECK   │
  └────┬─────┘
       ↓
  ┌──────────┐
  │   ACT    │
  └──────────┘
        ↻ (repeat)
```

| Phase | Action |
|-------|--------|
| **Plan** | Define the problem, analyze current state, set target, design countermeasure |
| **Do** | Implement the solution on a small scale / pilot |
| **Check** | Measure results against targets, analyze gaps |
| **Act** | Standardize if successful; otherwise adjust and repeat |

## Practical Template

| Phase | Questions to Answer |
|-------|-------------------|
| **Plan** | What's the current state? What's the target? What change will we test? |
| **Do** | Who does what? When? With what resources? |
| **Check** | Did we hit the target? What data supports this? What went wrong? |
| **Act** | Standardize or abandon? What's the next PDCA? |

## Example: Reduce Meeting Hours

| Phase | Action |
|-------|--------|
| Plan | Team avg 12 hrs/week → target 8 hrs. Test "no-meeting Wednesday" |
| Do | Block Wednesdays for 4 weeks, move standing meetings |
| Check | Week 4 avg = 7.5 hrs. Productivity survey +15%. |
| Act | **Standardize:** permanent no-meeting Wednesday policy |

## Best For
- Process improvement
- Quality management
- Behavior change at team/org level
- Kaizen (continuous improvement) initiatives
